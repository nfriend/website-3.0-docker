Location Manager is an intricate, complex software solution to the problem of large-scale, international research station data management. Unfortunately, due to the confidential nature of the application and the data it handles, little can be disclosed about LM's inner workings. Here are just a few of the high-level considerations that went into LM's architectural design:

  • Speed. LM has dependencies on several legacy systems, many of which have less than ideal response times. LM aggressively caches almost all the data it uses and relies heavily on lazy loading to optimize startup times.
  • Reliability. In addition to performance issues, many of these same legacy systems have a habit of going down at just the wrong time. Because LM is a mission-critical application, even moderate levels of volatility are unacceptable. Where possible, LM has been designed to run "disconnected" from these systems and allow end-users to maintain productivity.
  • Backwards compatibility. Location Manager will serve as a replacement for a number of legacy application within Pioneer's application ecosystem. LM maintains backwards compatibility where feasible and provides new alternatives for functionality slated for retirement.
  • Longevity. DuPont Pioneer understands Location Manager to be a long-term investment. Great pains have been taken to ensure that Location Manager will be maintainable and extensible for years after its release.
